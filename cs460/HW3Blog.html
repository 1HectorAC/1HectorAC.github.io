Step 4 (C# Changes):

Node.java to Node.cs

The big difference here was just setting up a property to get and set the data variable, which was changed to private. The code is pretty simple and using data it is as easy as calling the Data instead of directly with data.

QueueInterface.java to IQueue.cs

The big difference here was the file name change. In C# it was more proper to start interface title with I.

QueueUnderflowException.java to QueueUnderflowException.cs

Some differences here I came across where how to extend a class. Not much different with C# using a “:” instead of an “extends”. Calling the extended classes constructor was also a bit different with a different keyword and location to place it.

LinkedQueue.java to LinkedQueue.cs

This had a difference in implementing an interface with using “:” instead of “implements”. Another thing that I learned was the difference in exception types (or names of them) from java and C#. I had to look up a list of exception of C# to know what was available to use. This site was usefull in figuring out which to use :https://www.developerfusion.com/article/1889/exception-handling-in-c/3/.

I also noticed that in c# you can’t set a generic type to null unlike in java. This solution involved just setting the variable to default(type) instead of a null value. Like this:

                
T tmp = default(T);
Main.java to Program.cs

The first thing to note is that the file that was first generated by visual studio ended up being called Program instead of main so the file names ended up changing. One thing that was slightly different was the method names in their built in linked list. I also noticed that I couldn't find a method to directly return the element of the last node (like in the java code) but instead had to access the last node and had to access the element from there. I used this site as a reference for c# linkedlist: https://docs.microsoft.com/en-us/dotnet/api/system.collections.generic.linkedlist-1?view=netframework-4.7.2 Here is the diff: 

                
int maxLength = output.Last.Value.Length;
Vs
int maxLength = output.getLast().length();

Then there was a couple small syntaxial difference like using Console.WriteLine() instead of System.out.println() to outputted in command line and name changes that aren't really worth mentioning considering how trivial they are.

Some general things that were noted were xml comments for classes and functions. This was done by typing “///” a line above from the target. Then I just filled out the areas with descriptions of the specified area.
