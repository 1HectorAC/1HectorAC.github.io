Step 2 (Routes):


One of the first parts I worked on was the path routing. I pretty much just went to the App.js file and added the routes. The code was pretty simple. Example code of the routing is shown. Every time a new route is needed it just involves making another route element with a "path" and "element" fields. The "element" field in the code just refers to a components that is imported from another file and the "path" part is just the name of the path.


// Routes in App.js file.
import { BrowserRouter,Routes,Route } from 'react-router-dom';
import Page1 from './pages/Page1';
import Page2 from './pages/Page2';

function App() {
  return (
    <div className="App">
      <BrowserRouter>
        <Routes>
            <Route index element={<Page1/>} />
            <Route path="/Page2" element={<Page2/>} />
        </Routes>
      </BrowserRouter>
    </div>
  );
}

Step 3 (Navbar):


The next part I will mention is how the nav bar was setup. This part involves wrapping the routes in another "route" tag with a closing tag. This "route" tag will link to the component that contains the nav bar. In this case I called it "Layout" and its path is set to "/". Example shown.


// Navbar setup in App.js file.
<Routes>
    <Route path="/" element={<Layout/>}>
        <Route index element={<Page1/>} />
        <Route path="/Page2" element={<Page2/>}/>
    </Route>
</Routes>

I should also mention that the Layout page will also need a "<Outlet/>" tag at the end of the components return with its related library reference at the top.

Step 5 (Calculators):

There are 3 calcualtors on this website. The first is about calculating the saving from a percentage off sale. The second one involves calculating the tip cost based on a certain percentage. The final one is just understanding the savings from a buy 2 get 1 free. All the calculators have pretty similar code so I will just talk about the feature from the first one called "SaleCalculator."


The SaleCalculator will first take in two numbers, cost of the item and the percentage off. I setup hooks to control the state of these numbers and setup an "onchange" event to an input that control their value. The hook is shown at the top of the SaleCalculator component. I also added some number validation to restrict the value size. Below is some example code of what I have mentioned. The code is only for the cost but is nearly identical for percentage off. 


// Select parts of SaleCalculator.js
import { useState } from "react";

const SaleCalculator = () => {
    const [cost, setCost] = useState(0);
    
    function changeCost(e) {
        let input = Number(e.target.value);
        if (input > 1000000)
            e.target.value = input = 1000000;
        else if (input < 0)
            e.target.value = input = 0;
        setCost(input);
    }
    return(
        <div>
            // Other code
            <input type="number" onChange={changeCost} />
            // Display results section
        </div>
    )
}

The final part is to display the results. With the "cost" and "percentage" variables set as a state I can reference the variables on the page and if the variables get update then the reference that is display also get updated. Note that there variables can only be reference in the html part with curly brackets. No I can just add the variables and calculations at the end of the page after the input section. I also restricted the length of the number to 2 decimal places to make it easier to see using the toFixed method. Example shown:


// Display result section in SaleCalculator.
<p>(Cost - Saving = New Cost)</p>
{cost.toFixed(2)} - {(percentage / 100 * cost).toFixed(2)} = {(cost - percentage / 100 * cost).toFixed(2)}
